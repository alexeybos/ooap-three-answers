Создание объектов:
- После запуска приложения (Запускается jar-файл).
- GameEngine создает CommandHandler
(На экране приветствие и предложение начать игру "/start")
- GameEngine обрабатывает команду и создает GameField
- GameEngine инициирует создание объектов GameFieldElement для заполнения GameField
- GameEngine создает ScoreCounter, GameStatistics и HistoryWriter
- еще я понял, что мне не хватает FigureMatcher - класс для поиска/сравнения комбинаций. Также создается GameEngine

События:
для оповещения компонентов программы используется паттерн Observer.
GameEngine генерирует события - EventObserver подписывается на данные события.
Обработка хода: UserTurnCommand передается в CommandHandler - проверка валидности - оповещение ScoreCounter, GameStatistics и HistoryWriter.

По фреймворку не совсем уверен, как правильно тут описывать - используется Java и, т.к. по сути игра не имеет GUI, таким образом (по ТЗ) 
в консоль выводится сетка (игровое поле) просто с помощью System.out.println (обозначение элементов сетки символами (буквы A, B, C, D, E)). 
Ожидание ввода координат игрока. Внутренняя обработка команды, генерация измененного поля, отображение итогового поля (System.out.println) или информация об окончании игры.
HistoryWriter записывает историю в формате JSON с помощью библиотеки Jackson.

Типовые тестовые сценарии
Пользователь запускает игру - engine заполняет поле - отрисовывает поле на экране - ожидает хода пользователя.

Пользователь указывает координаты хода - валидация координат (указаны соседние поля) - валидация комбинации - убирается комбинация - добавляются очки - перезаполняется поле - перерисовывается поле.
1. Начало игры. Запуск приложения - приветственное сообщение и указание возможных команд - ввод команды /start - отображение заполненного поля - на поле нет "собранных" комбинаций - существует как минимум один ход, дающий комбинацию - ожидание ввода игрока.
2. Ход, не порождающий комбинацию (невалидный). Игрок указывает координаты "откуда - куда" - engine проверяет ход (ход не валиден) - ход отменяется - отображается поле в виде, предшествующем ходу игрока.
3. Успешный ход с одной комбинацией. Игрок указывает координаты "откуда - куда" - engine проверяет ход (ход валиден) - собранная комбинация удаляется - начисляются очки - игровые элементы "падают" на освободившиеся места - сверху падают новые элементы - engine проверяет наличие возможных потенциальных комбинаций (ходов нет) - ожидание следующего хода игрока
4. Успешный ход, порождающий несколько комбинаций. Игрок указывает координаты "откуда - куда" - engine проверяет ход (ход валиден) - собранная комбинация удаляется - начисляются очки - игровые элементы "падают" на освободившиеся места - сверху падают новые элементы - проверка наличия новых комбинаций и повторение (удаление комбинации - падение новых элементов) пока поле не успокоится - engine проверяет наличие возможных потенциальных комбинаций (ходов нет) - ожидание следующего хода игрока
5. Завершение игры при отсутствии ходов. После очередного хода engine проверяет наличие возможных потенциальных комбинаций (ходов нет) - игра завершается
6. Обновление статистики. Игрок делает результативный ход - в статистику записывается количество очков, увеличивается счетчик ходов.
7. Обновление истории. Игрок делает ход - в историю ходов добавляется сделанный ход.
